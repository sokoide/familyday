## 完了したこと

ブラウザで動く5分クイック体験「AI魔法のゲームブック『ドラゴン城の秘宝』(2026)」を `./2026/` 配下に新規実装した。

### 構成
- **バックエンド**: `2026/server/`(Go・Clean Architecture 4層)
  - `internal/domain/` 値オブジェクト(Verdict/Lives/EndingType/DragonRoute/Lang/Stage)+ 純粋ルール
  - `internal/usecase/` JudgeUseCase/EndingUseCase + ポート定義
  - `internal/adapters/presentation/http/` handler/DTO/エラー→status変換
  - `internal/adapters/infra/` gemini(Gemini判定+Imagen画像)/persistence/ratelimit/sysid
  - `internal/app/` Composition Root のワイヤリング(BuildMux)。cmd と統合テストで再利用
  - `cmd/server/main.go` 起動エントリ(env読込→BuildMux→HTTPサーバ)
- **フロント**: `2026/web/`(TypeScript + Vite)
  - `app`(状態機械) / `ports`(抽象) / `infra`(fetch・WebSpeechの具象) / `ui`(qr/share)
  - i18n: `app/messages.ts` にUI文字列集約。日/英切替(開始画面・localStorage保持)
  - 音声入力は Web Speech API、フォールバックの手動入力あり
  - QRコード + メール送信(mailto:・アドレスはサーバへ送らない)
- **Makefile**(ルート): build / unit / integration / vet / fmt / check / run / dev / clean / help
- **README**: Mermaid図入りで起動・アーキテクチャ・API・デプロイ・テスト・i18nを網羅

### テスト・品質(すべて緑)
- `make` / `make unit`: ユニットテスト(domain/usecase/presentation)・ネットワーク不要
- `make integration`: 実Gemini/Imagen のE2E(`//go:build integration`・GEMINI_API_KEY無しはskip)
- `make vet` / `make check`(cleanarch): PASS
- 実APIで ja/en の判定メッセージ・ストーリー生成・画像生成を確認済み

### 設計上の判断点(記録)
- ライフ权威はフロント保持(サーバはステートレス・3並列でセッション管理不要)。→ `/api/ending` の cleared はクライアント権威なので捏造可能。閉鎖イベントのため許容。
- Gemini セーフティブロック/空応答は `Bad`(リトライ)に倒す(体験継続優先)。真の通信・認証エラーは upstream(502)。
- モデル名は `gemini.Config` + 環境変数(`GEMINI_MODEL_JUDGE/STORY/IMAGEN`)で差し替え可能。
- Imagen 失敗時はフロント `onerror` で data-URI fallback 画像へ。
- 来年は `2026/` を `2027/` にコピーし、`domain/catalog.go` と `web/src/app/messages.ts` を差し替えれば再利用可能。

## よく使うコマンド

```bash
make                  # build + unit(既定)
make integration      # 実APIのE2E(GEMINI_API_KEY 必要)
make run              # サーバ起動(2026/.env を読む)
make dev              # フロント開発サーバ
make check            # Clean Architecture チェック
```

ローカル起動:
```bash
cd 2026/web && npm install && npm run build   # 初回のみ
cd 2026/server && cp ../.env.example ../.env   # GEMINI_API_KEY 記入
make run   # http://localhost:8080
```
